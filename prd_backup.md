1. 프로젝트 개요 (Project Overview)
서비스명 (가칭): 포폴링크 (PoFoLink)

타겟 고객: 이력서(텍스트)와 활동 내역(사진)은 있지만, 이를 시각적인 포트폴리오로 구현할 디자인/개발 역량이나 시간이 없는 IT 직군 취준생 및 대학원 진학 준비생.

해결하고자 하는 문제 (Pain Points):

기존 PDF 포트폴리오를 제작하는 데 수 주일의 시간과 불필요한 에너지가 소요됨.

업데이트 시 매번 파일을 수정하고 재전송해야 하는 번거로움.

핵심 가치 (Core Value):

유저가 이력서 PDF와 사진만 업로드하면, AI가 데이터를 분류하고 기존에 완성된 고퀄리티 템플릿(GitHub 코드)에 자동 주입하여 5분 만에 '웹 포트폴리오 URL'을 생성해 줌.

2. 사용자 시나리오 (User Flow)
MVP(최소 기능 제품) 기준, 사용자가 진입해서 결과물을 얻기까지의 핵심 동선입니다.

진입 & 업로드: 랜딩 페이지 접속 ➡️ "이력서로 웹 포트폴리오 만들기" 클릭 ➡️ 본인의 이력서(PDF) 업로드.

AI 자동 분석 (10초 대기): 시스템이 이력서 텍스트를 추출하고, [기본 정보], [기술 스택], [프로젝트 경험] 등으로 구조화.

스마트 매칭 (유저 개입 5분):

화면에 AI가 채워둔 텍스트 폼이 나타남.

유저가 텍스트를 가볍게 검수/수정함.

각 프로젝트 항목 옆에 있는 '사진 추가' 버튼을 눌러 해당 프로젝트에 맞는 이미지를 업로드 (텍스트-이미지 매칭 완료).

미리보기: 입력이 완료되면 우측 화면(또는 다음 페이지)에서 내 데이터가 들어간 실제 웹사이트 템플릿을 미리 확인.

발행 (배포): 'URL 생성하기' 클릭 ➡️ username.pofolink.com 형태의 고유 링크가 발급되며 즉시 사용 가능.

3. 핵심 기능 목록 (Core Features)
PDF 텍스트 추출 및 구조화 (AI): 업로드된 이력서 파일에서 순수 텍스트를 뽑아내고, 이를 AI를 통해 서비스에 맞는 정형화된 JSON 데이터로 변환.

스마트 입력 폼: AI가 1차로 채워준 데이터를 유저가 직관적으로 수정하고, 특정 섹션(프로젝트 등)에 이미지를 직접 첨부할 수 있는 UI.

템플릿 렌더링 엔진: 유저가 입력 완료한 데이터(JSON)를 미리 준비된 프론트엔드 뼈대 코드(Template)에 주입하여 화면을 그려내는 기능.

자동 호스팅 및 도메인 발급: 생성된 결과물을 별도의 서버 세팅 없이 즉시 웹에 배포하고 서브 도메인을 매핑하는 인프라 연동.

4. 기술 스택 (Tech Stack)
직장인 2인 체제의 리소스를 고려하여 초기 비용 0원, 서버리스(Serverless) 구조로 설계합니다.

프론트엔드 (UI & 템플릿): Next.js (React 기반, 뼈대 코드 이식 및 SEO 최적화), Tailwind CSS (빠른 스타일링), Zustand (상태 관리).

백엔드 & 데이터베이스 (BaaS): Supabase (유저 데이터 및 JSON 저장용 PostgreSQL, 이미지 원본 저장용 Storage).

AI & 파이프라인: pdf-parse (PDF 텍스트 추출), OpenAI API (GPT-4o-mini) (텍스트 구조화 및 요약).

인프라 & 배포: Vercel (자동 빌드/배포 및 Wildcard Subdomain 라우팅).

5. 화면 구성 (Screen Layout)
MVP 출시에 필수적인 핵심 화면 3가지입니다.

랜딩 및 업로드 페이지:

서비스 소개 한 줄 요약 및 예시 웹사이트 링크.

중앙에 커다란 [PDF 파일 드롭 존] 및 [분석 시작] 버튼.

워크스페이스 (데이터 폼 & 매칭 페이지):

화면 좌측: [기본 정보], [핵심 역량], [프로젝트 1, 2, 3] 등 아코디언 형태로 열리는 입력 폼. (AI가 채워둔 상태)

프로젝트 폼 내부: 제목/설명 텍스트 입력 칸 + [이 프로젝트 썸네일 업로드] 버튼.

최종 결과 및 URL 발급 페이지:

생성 완료된 전체 웹사이트 화면 (미리보기).

하단에 생성된 내 고유 링크 표기 및 [링크 복사하기] 버튼.

6. 상세 기능 명세 (자동화 로직 핵심)
개발자님이 가장 강조하신 **"어떻게 텍스트와 사진이 매칭되어 뼈대 코드에 꽂히는가?"**에 대한 기술 명세입니다.

Step 1. AI 데이터 파싱 (JSON 생성):

OpenAI API를 통해 이력서 텍스트를 아래와 같은 구조의 JSON으로 반환받습니다.

{ "name": "김철수", "skills": ["React", "Node.js"], "projects": [ { "id": 1, "title": "쇼핑몰 구축", "desc": "결제 시스템 연동" } ] }

Step 2. 이미지 매핑 (JSON 업데이트):

유저가 2번 화면(워크스페이스)에서 특정 프로젝트(id: 1)에 사진을 업로드합니다.

이미지는 Supabase Storage에 저장되고 URL을 반환받습니다.

해당 URL이 JSON에 추가됩니다. ➡️ "projects": [ { ..., "image_url": "https://supabase.com/..." } ]

Step 3. 뼈대 코드 주입 (Props Injection):

친구분이 만든 Next.js 템플릿 페이지(PortfolioTemplate.tsx)는 내부적으로 하드코딩된 글씨가 아니라, 외부에서 JSON 데이터를 Props로 받아 화면을 그리도록 설계되어야 합니다.

최종 완성된 JSON 덩어리를 템플릿 컴포넌트에 넘기면, 코드가 알아서 <img src={data.projects[0].image_url} />, <h1>{data.name}</h1> 형식으로 렌더링하여 완전한 웹사이트로 구워냅니다.

7. 디자인 가이드 (Design Guidelines)
입력 폼 (Admin/User UI):

흰색/밝은 회색 배경의 깔끔한 대시보드 스타일. 사용자가 '입력과 수정'이라는 목적에만 집중할 수 있도록 화려한 요소를 배제합니다.

결과물 템플릿 (Portfolio UI):

기존에 친구분이 완성해 둔 GitHub 뼈대 코드의 디자인 시스템을 100% 존중하여 사용합니다.

가능하다면 지원자의 개성을 살릴 수 있도록 (코드 단에서) 테마 색상(Primary Color) 정도만 변경할 수 있는 변수 처리를 권장합니다.

8. 제약 사항 (Constraints & Fallbacks)
업로드 제한: 무분별한 스토리지 비용 증가를 막기 위해 유저당 업로드 가능한 총 이미지 용량을 20MB(장당 5MB 이하)로 제한하고, 업로드 시 브라우저 단에서 리사이징/압축(예: browser-image-compression 활용)을 강제합니다.

AI 할루시네이션(오류) 대처: 이력서 형식이 난해하여 AI가 파싱을 망칠 경우를 대비해, 유저가 워크스페이스 폼에서 내용을 직접 지우고 새로 쓸 수 있도록 완전한 편집 권한을 열어두어야 합니다.

초기 수익화 및 워터마크: 완전 무료로 제공하되, 생성된 포트폴리오 최하단에는 서비스 홍보를 위한 워터마크("포폴링크로 5분 만에 제작됨")를 고정 삽입하여 바이럴을 유도합니다.
